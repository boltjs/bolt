Use

There are multiple aspects to using bolt:

  • elements of a bolt project
  • configuring a project
  • running up a project
  • defining modules
  • compiling modules
  • depending on modules

Elements of a bolt project

Get bolt,. 

$ dent -clean

Initialise bolt: 

$ ./lib/bolt/bolt init

This will create a bolt/config directory in your project, initialise an empty module.js configuration file, explained below, and also run the `jsc dev` and `jsc mode dev` commands to generate an initial bootstrap.js file and set the project to development mode.

For convenience, you may wish to define an alias for the bolt and jsc executables. This has already been done for development machines via the default ephox-bashrc in buildtools/scripts.

Configuration

Bolt is configured through via the configure api call. By convention this call is placed in a configuration file named bolt/config/module.js.

The configure call takes an array of modulators, for example: 

configure({
  modulators: [
    modulator('js', 'ephox.modulator.js'),
  ],
  sources: [
    source('amd', 'ephox.myproject', '../../src/main/js'),
    source('js', 'lib', '../../lib/compile')
  ]
});

Getting Running

Upon check out of a project using bolt, there a small set of tasks that are required to get up and running. These tasks are wrapped up by the bolt script. 

$ bolt init

This command will generate a bootstrap file and create a symlink ready for development.
